Multi-Source Company Lead Generator

A flexible Python-based lead generation tool that extracts company information and employee LinkedIn profiles from any URL - job boards, portfolio pages, company directories, or direct company websites.

🎯 What This Tool Does

  1. Multi-Source Scraping - Process unlimited URLs from Excel input
  2. Intelligent URL Detection - Automatically classifies: listing pages, profile pages, or direct companies
  3. Company Discovery - Extracts company domains from any source
  4. Employee Enrichment - Finds LinkedIn profiles + Apollo.io emails
  5. Excel Output - Multi-sheet reports with source tracking
  6. Blacklist Support - Skip unwanted companies

📋 Features

  • Multi-Source Input: Excel file with unlimited URLs to scrape
  • Auto URL Classification: Detects listing pages, profile pages, or direct companies
  • Generic Extraction: Works with any job board, portfolio page, or website
  • Jina API Integration: Clean markdown extraction from any URL
  • Google Custom Search: Finds LinkedIn profiles via Google's search API
  • Apollo.io Integration: Automatic email enrichment with credits
  • Smart Filtering: Excludes job boards, social media, and platform domains
  • Confidence Scoring: Rates extraction quality for each company
  • Source Tracking: Know where each lead came from
  • Person Identification: Unique IDs for deduplication across sources
  • Blacklist System: Skip unwanted companies across all sources
  • Excel Output: Multi-sheet reports (Companies, People, Summary)

🚀 Complete Setup Guide (For Beginners)

Step 1: Install Python

Make sure you have Python 3.7+ installed:

python3 --version

If not installed, download from python.org

Step 2: Clone/Download This Repository

git clone <your-repo-url>
cd "Lead Generator"

Step 3: Create Virtual Environment

# Create virtual environment
python3 -m venv venv

# Activate it (macOS/Linux)
source venv/bin/activate

# Activate it (Windows)
venv\Scripts\activate

Step 4: Install Dependencies

pip install -r requirements.txt

Step 5: Get API Keys

You need two API keys (both FREE):

A. Jina Reader API Key (FREE)

  1. Visit jina.ai/reader
  2. Sign up for free account
  3. Get your API key (starts with jina_...)

B. Google Custom Search API (FREE - 100 searches/day)

Create Google Cloud Project:

  1. Go to Google Cloud Console
  2. Create a new project (any name, e.g., "Lead Generator")
  3. Note your Project ID

Enable Custom Search API:

  1. In Cloud Console, go to APIs & Services > Library
  2. Search for "Custom Search API"
  3. Click on it and press Enable

Create API Key:

  1. Go to APIs & Services > Credentials
  2. Click Create Credentials > API Key
  3. Copy the key (starts with AIza...)
  4. (Optional) Click Restrict Key and limit to Custom Search API only

Create Custom Search Engine:

  1. Go to Programmable Search Engine
  2. Fill in:
    • Search engine name: "LinkedIn People Search" (or any name)
    • What to search: Select "Search specific sites or pages"
    • Sites to search: Enter linkedin.com/in/*
  3. Click Create
  4. On the next page, copy your Search Engine ID (looks like e43f38264dff14931)

Important: The free tier gives you 100 searches/day. After that, it's $5 per 1,000 queries.

Step 6: Create .env File

Create a file named .env in the project root:

# In project directory
touch .env

Add your API keys to .env:

# Jina Reader API Key
JINA_API_KEY=jina_your_actual_key_here

# Google Custom Search API credentials
GOOGLE_API_KEY=AIza_your_actual_key_here
GOOGLE_CSE_ID=your_search_engine_id_here

⚠️ Important:

  • Replace the values with YOUR actual keys
  • Never commit .env to git (it's already in .gitignore)
  • Keep your keys secret!

Step 7: Create Your Source URLs File (Optional)

The scraper can handle multiple URLs from an Excel file:

Option A: Use Template (Recommended)

# Copy the template
cp sources_template.xlsx sources.xlsx

# Open in Excel and edit with your URLs

Option B: Create From Scratch

Create sources.xlsx with these columns:

source_url source_name url_type
https://jobs.a16z.com/jobs A16Z Portfolio auto
https://www.ycombinator.com/jobs YC Companies auto
https://techcorp.com Tech Corp direct_company

Column Descriptions:

  • source_url (required): URL to scrape
  • source_name (optional): Friendly name for reports
  • url_type (optional): auto, listing_page, profile_page, or direct_company

URL Types Explained:

  • auto - Let the scraper detect (recommended)
  • listing_page - Multiple companies listed (e.g., job boards, portfolios)
  • profile_page - Single company page (e.g., company job page)
  • direct_company - Actual company website

Note: If no sources.xlsx exists, scraper uses default A16Z URL.

Step 8: Run the Scraper

# Make sure venv is activated
source venv/bin/activate

# Run the scraper
python3 lead_scraper.py

The script will:

  1. Load blacklist (if exists)
  2. Load sources from sources.xlsx (or use default A16Z)
  3. Process each source URL
  4. Auto-classify URL type
  5. Extract companies
  6. Find LinkedIn profiles + Apollo emails
  7. Save results to leads_YYYY-MM-DD_HH-MM-SS.xlsx

🚫 Blacklist System

To skip specific companies:

  1. Open blacklist.csv
  2. Add company URLs (one per line):
company_url
https://company-to-skip.com
https://another-company.com/

The scraper will:

  • Load blacklist at startup
  • Skip blacklisted companies entirely
  • Not search LinkedIn for them
  • Not include them in output

Notes:

  • URLs are case-insensitive
  • Trailing slashes are ignored
  • Blacklist is optional (can be empty)

⚙️ Configuration

Edit these settings in lead_scraper.py (lines 34-61):

Change Max Profiles Per Company

# Line 34
MAX_PROFILES_PER_COMPANY = 10  # Change to 20, 50, etc.

Modify Job Titles to Search

# Lines 38-54 - Add/remove titles
SEARCH_JOB_TITLES = [
    "Founder", "CEO", "CTO",  # Add your titles here
    # ...
]

Toggle Current Employee Filter

# Line 57
FILTER_CURRENT_EMPLOYEES = True  # Set False to include alumni

Adjust Rate Limiting

# Line 60
API_RATE_LIMIT_DELAY = 1.0  # Increase if hitting rate limits

Change Target URL

# Line 484 in main() function
jobs_url = "https://your-target-website.com/jobs?filters=..."

You can customize the URL to scrape from any job board or company listing page that returns company profiles.

🔍 How It Works

  1. Fetch Listings Page - Gets job board page via Jina API (converts to markdown)
  2. Extract Company Profiles - Uses regex patterns to find company links
  3. Get Company URLs - Visits each profile to extract actual website URLs
  4. Check Blacklist - Skips if company URL is blacklisted
  5. LinkedIn Search - Searches Google for employee LinkedIn profiles
    • Query: site:linkedin.com/in/ "CompanyName" "current" (title1 OR title2...)
    • Finds up to 10 profiles per company
  6. Extract Profile Data - Gets name, title, LinkedIn URL from results
  7. Save to CSV - Flattens to one row per person

📈 API Usage & Limits

Jina API

Google Custom Search API

🐛 Troubleshooting

"No module named 'requests'" or similar

# Make sure venv is activated
source venv/bin/activate
pip install -r requirements.txt

"Google API credentials not found"

  • Check .env file exists in project root
  • Verify GOOGLE_API_KEY and GOOGLE_CSE_ID are set
  • Make sure no spaces around = sign

"Rate limit exceeded" or "Quota exceeded"

  • Google free tier is 100 searches/day
  • Wait 24 hours or upgrade to paid tier
  • Reduce MAX_PROFILES_PER_COMPANY to use fewer API calls

"No results returned from Google"

  • Company name might not match LinkedIn exactly
  • Try adjusting SEARCH_JOB_TITLES list
  • Check Google Custom Search Engine settings

Empty CSV or no people found

  • Verify Google API credentials are correct
  • Check company names are spelled correctly
  • LinkedIn profiles might be private/not indexed
  • Verify the target URL is accessible and returns expected format
